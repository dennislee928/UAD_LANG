# UAD 語言品牌指南

本文檔定義了 UAD Programming Language 的品牌視覺識別系統。

## 品牌概述

**UAD (Unified Adversarial Dynamics)** 是一門專為對抗動態建模、道德風險量化和認知安全系統設計的領域專用語言。

### 核心價值

- **專業性**: 面向專業的系統建模和安全工程
- **創新性**: 融合音樂理論、弦理論等跨領域概念
- **可靠性**: 強型別系統確保模型正確性

## Logo 規範

### Logo 設計原則

1. **簡潔性**: 保持設計簡潔，易於識別
2. **技術感**: 傳達程式語言的技術屬性
3. **獨特性**: 與其他語言 Logo 有明顯區別
4. **可擴展性**: 在不同尺寸下都能清晰顯示

### Logo 尺寸要求

| 用途 | 尺寸 | 格式 | 說明 |
|------|------|------|------|
| GitHub Org Avatar | 256×256px | PNG | 方形，透明背景 |
| Social Preview | 1200×630px | PNG/JPG | 橫幅，GitHub 顯示 |
| Extension Icon | 128×128px | PNG | 多尺寸：16, 32, 48, 64, 128 |
| Favicon | 32×32px | ICO | 瀏覽器標籤 |
| 文件內嵌 | 200×200px | SVG/PNG | 文件中的 Logo |

### Logo 使用場景

#### 1. GitHub Organization Avatar

- **尺寸**: 256×256px
- **格式**: PNG（透明背景）
- **位置**: GitHub Organization 設定
- **文件名**: `uad-org-avatar-256.png`

#### 2. Repository Social Preview

- **尺寸**: 1200×630px
- **格式**: PNG 或 JPG
- **位置**: Repository Settings → Social preview
- **文件名**: `uad-social-preview-1200x630.png`
- **內容**: Logo + 簡短文字描述

#### 3. VS Code Extension Icon

- **尺寸**: 
  - 16×16px
  - 32×32px
  - 48×48px
  - 64×64px
  - 128×128px
- **格式**: PNG
- **位置**: `uad-vscode/icons/`
- **命名**: `uad-icon-{size}.png`

#### 4. Favicon

- **尺寸**: 16×16px, 32×32px
- **格式**: ICO（包含多尺寸）
- **位置**: Landing Page
- **文件名**: `favicon.ico`

#### 5. 文件 Logo

- **尺寸**: 可縮放（建議 200×200px）
- **格式**: SVG（優先）或 PNG
- **位置**: README.md, 文件頁面
- **文件名**: `uad-logo.svg` 或 `uad-logo-200.png`

## 使用指南

### Logo 使用規則

1. **保持比例**: 不要拉伸或壓縮 Logo
2. **最小尺寸**: 確保 Logo 不小於最小建議尺寸
3. **背景**: 在淺色和深色背景下都要清晰可見
4. **間距**: Logo 周圍保持足夠的空白空間

### 顏色使用規則

1. **主色調優先**: 優先使用主要藍色 `#4A90E2`
2. **對比度**: 確保文字和背景有足夠對比度（WCAG AA 標準）
3. **一致性**: 在所有平台上使用相同的顏色值

### 禁止事項

1. ❌ 不要修改 Logo 的顏色或比例
2. ❌ 不要在 Logo 上添加其他元素
3. ❌ 不要使用低解析度的 Logo
4. ❌ 不要將 Logo 用於未授權的用途
